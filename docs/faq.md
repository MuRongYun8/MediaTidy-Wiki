# 常见问题

## 部署相关

### 启动后无法访问管理面板？

1. 确认端口映射正确：`docker compose` 中的 ports 配置
2. 检查防火墙是否放行对应端口
3. 查看日志：`docker logs mediatidy`
4. 确认 `LICENSE_CODE` 已正确配置

### 授权验证失败？

- 检查 `LICENSE_CODE` 环境变量是否正确设置
- 确认容器可以访问公网（授权验证需要联网）
- 如配置了代理，检查代理是否正常
- 查看日志中的具体错误信息

### Docker 容器不断重启？

MediaTidy 在授权失败时会保持进程存活（不退出），但 HTTP 服务不会启动。查看日志确认原因。

## 功能相关

### 整理功能不可用？

自动整理依赖 TMDB API Key。检查：
1. 「系统设置」中是否已配置 TMDB API Key
2. API Key 是否有效（可在 TMDB 官网测试）
3. 网络是否可达 api.themoviedb.org（可能需要代理）

### 文件浏览器显示为空？

1. 确认 CD2 已正确配置并可连接
2. 确认 115 账号 Cookie 有效
3. 确认路径映射已添加
4. 检查日志中是否有 CD2 连接错误

### STRM 文件生成后 Emby 看不到？

1. 确认 STRM 输出目录已添加到 Emby 媒体库
2. 手动触发 Emby 媒体库扫描
3. 检查 STRM 文件内容是否正确（应为播放地址 URL）
4. 确认 Emby 的「实时监控」已开启

### 整理后文件名不对？

检查命名规则配置：
1. 进入「规则引擎」查看当前命名规则
2. 确认模板语法正确
3. 在「工具箱」中测试文件名识别结果

### 115 频繁触发风控？

1. 增大 API 调用延迟（网盘配置 → 延迟配置）
2. 减少并发任务数量
3. 开启风控防护自动破解
4. 避免短时间内大量操作

## 性能相关

### 内存占用过高？

1. 减小 `GOMEMLIMIT`（默认 512MiB）
2. 减少整理任务的并发数
3. 减小 TMDB 缓存的最大条目数
4. 避免同时运行多个大型任务

### 整理速度慢？

1. 如果已配置 FUSE，文件访问引擎优先选「本地 FUSE」
2. 没有配置 FUSE 时，确认 CD2 gRPC 连接稳定
3. 适当增大并发数（注意风控风险）
4. 确认 TMDB API 响应正常（可能需要代理）
5. 检查 CD2 和 FUSE 挂载是否稳定

### STRM 生成速度慢？

1. 如果源目录是 115 网盘，数据源优先选「115 高速穿透」
2. 超大目录首次生成可以考虑「115 目录树导出」
3. 适当提高 STRM 并发数，但如果触发 115 风控就降回来
4. 确认 115 Cookie 有效，且账号没有处于风控状态

### 如何备份数据？

备份 `data` 目录即可，包含：
- `mediatidy.db`：数据库（配置、任务历史、规则等）
- `logs/`：日志文件

### 如何查看当前版本？

- 管理面板左下角
- 启动日志：`MediaTidy starting version=x.x.x`

### 支持哪些平台？

Docker 镜像支持：
- `linux/amd64`（x86_64）
- `linux/arm64`（aarch64，如树莓派 4、群晖 ARM）
