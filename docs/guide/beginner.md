# 新手须知

这是一份给第一次部署 MediaTidy 的傻瓜式流程。你不需要先理解所有高级功能，只要按顺序完成下面几步，就能把服务跑起来，并完成第一次自动整理或 STRM 生成。

::: tip 建议
新手先只做一件事：让 MediaTidy 能看到你的网盘目录，并成功处理一个测试文件。确认链路正常后，再开启定时任务、实时监听、Emby 反代、通知等高级功能。
:::

## 先看懂整体关系

MediaTidy 不是单独替代网盘、CD2 或 Emby，它更像一个自动化中控。

| 组件 | 做什么 | 新手是否必需 |
|------|--------|--------------|
| MediaTidy | 管理面板、自动整理、STRM 生成、缓存、任务队列 | 必需 |
| CloudDrive2 | 连接网盘，提供文件列表和文件操作能力 | 必需 |
| 115 网盘账号 | 存放媒体文件，提供直链和文件管理能力 | 推荐 |
| TMDB | 识别电影、电视剧信息 | 自动整理必需 |
| Emby / Jellyfin | 播放整理后的媒体库或 STRM 文件 | 可选但常用 |

最常见的新手链路是：

```text
115 网盘 -> CloudDrive2 挂载 -> MediaTidy 整理 / 生成 STRM -> Emby 扫描播放
```

## 准备材料

部署前先准备好这些东西，后面照着填就行。

| 需要准备 | 示例 | 用途 |
|----------|------|------|
| 一台运行 Docker 的机器 | NAS / Linux 服务器 / Windows Docker Desktop | 运行 MediaTidy |
| MediaTidy 授权码 | `LICENSE_CODE=xxxx` | 启动服务 |
| CloudDrive2 地址 | `http://192.168.1.100:19798` | 连接 CD2 |
| CloudDrive2 账号密码或 Token | 自己的 CD2 登录信息 | 调用 CD2 |
| 115 Cookie | 浏览器登录 115 后复制 | 115 原生 API |
| 媒体源目录 | `/我的115/downloads` | 待整理或生成 STRM 的目录 |
| 媒体目标目录 | `/我的115/媒体库` | 整理后的目录 |
| STRM 输出目录 | `/app/strm` | 输出给 Emby 扫描 |
| Emby 地址和 API Key | `http://192.168.1.100:8096` | 刷新媒体库、反代播放 |

::: warning 注意
路径一定要先想清楚。新手最容易卡在“宿主机路径、容器路径、CD2 挂载路径、MediaTidy 虚拟路径”混在一起。本文后面会按最简单的一套路径来配置。
:::

## 第一步：安装 CloudDrive2

MediaTidy 需要通过 CloudDrive2 看到你的网盘文件。

1. 安装并启动 CloudDrive2。
2. 登录 CloudDrive2 管理界面。
3. 添加 115 网盘账号。
4. 确认能在 CD2 里看到你的网盘目录。
5. 记录 CD2 的服务地址，例如 `http://192.168.1.100:19798`。

如果你使用 FUSE 挂载，建议把网盘挂载到宿主机固定目录，例如：

```text
/CloudNAS
```

验收标准：

- CD2 管理界面能正常打开。
- CD2 里能看到 115 网盘目录。
- 记下了 CD2 地址、账号密码或 Token。

## 第二步：创建部署目录

在服务器上找一个固定目录存放 MediaTidy 数据，例如：

```bash
mkdir -p /opt/mediatidy
cd /opt/mediatidy
mkdir -p data strm
```

目录用途：

| 目录 | 用途 |
|------|------|
| `/opt/mediatidy/data` | 保存数据库、配置、日志 |
| `/opt/mediatidy/strm` | 保存生成的 `.strm` 文件 |
| `/CloudNAS` | CD2 的 FUSE 挂载目录 |

::: tip Windows / NAS 用户
如果你的系统不是 Linux，把示例中的 `/opt/mediatidy` 换成你自己的实际目录即可。关键是 `data` 和 `strm` 要长期保留，不要放在临时目录。
:::

## 第三步：写 docker-compose.yml

在部署目录中新建 `docker-compose.yml`：

```yaml
services:
  mediatidy:
    image: murongyun574/mediatidy:latest
    container_name: mediatidy
    restart: unless-stopped
    ports:
      - "2019:2019"
    volumes:
      - ./data:/app/data
      - ./strm:/app/strm
      - /CloudNAS:/CloudNAS:rslave
    environment:
      - TZ=Asia/Shanghai
      - LICENSE_CODE=填入你的授权码
```

字段怎么理解：

| 配置 | 说明 |
|------|------|
| `2019:2019` | 管理面板端口，浏览器通过这个端口访问 |
| `./data:/app/data` | 持久化数据，升级容器也不会丢配置 |
| `./strm:/app/strm` | STRM 输出目录 |
| `/CloudNAS:/CloudNAS:rslave` | 把 CD2 FUSE 目录挂进容器 |
| `LICENSE_CODE` | MediaTidy 授权码 |

如果你没有使用 CD2 FUSE，可以先删掉这一行：

```yaml
- /CloudNAS:/CloudNAS:rslave
```

## 第四步：启动 MediaTidy

在 `docker-compose.yml` 所在目录执行：

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

查看启动日志：

```bash
docker logs -f mediatidy
```

看到没有持续报错后，打开浏览器访问：

```text
http://服务器IP:2019
```

第一次进入需要创建管理员账号。

验收标准：

- 浏览器能打开 MediaTidy 管理面板。
- 能注册并登录管理员账号。
- `docker logs -f mediatidy` 没有授权失败或端口占用错误。

## 第五步：配置 CloudDrive2

进入 MediaTidy 管理面板后，先配置 CD2。

1. 打开「网盘配置」。
2. 找到「CloudDrive2 配置」。
3. 填入 CD2 地址，例如 `http://192.168.1.100:19798`。
4. 选择认证方式：用户名密码或 Token。
5. 如果使用 FUSE，填入 FUSE 挂载路径，例如 `/CloudNAS`。
6. 保存并测试连接。

常见填法：

| 配置项 | 示例 |
|--------|------|
| CD2 地址 | `http://192.168.1.100:19798` |
| 认证方式 | `password` |
| FUSE 挂载路径 | `/CloudNAS` |

验收标准：

- CD2 连接测试通过。
- 文件管理器能看到 CD2 里的目录。

## 第六步：添加 115 账号

115 账号用于直连 API、路径映射、下载地址生成和风控处理。

1. 浏览器登录 [115.com](https://115.com)。
2. 按 `F12` 打开开发者工具。
3. 切到「网络」或「Network」。
4. 刷新 115 页面。
5. 点击任意 115 请求。
6. 在请求头里找到 `Cookie`。
7. 复制完整 Cookie。
8. 回到 MediaTidy 的「网盘配置」。
9. 添加 115 账号，填入账号名称和 Cookie。
10. 保存后等待系统验证。

::: warning 注意
Cookie 相当于登录凭证，不要发到公开群里。Cookie 过期后，回到这里重新复制并更新即可。
:::

验收标准：

- 115 账号状态为正常。
- 系统能获取到 115 用户信息。

## 第七步：配置路径映射

路径映射的作用是把 CD2 里的真实挂载路径，转换成 MediaTidy 里更好用的虚拟路径。

假设你的 CD2 里 115 挂载路径是：

```text
/115open
```

你添加的 115 账号名称是：

```text
我的115
```

那么可以配置：

| 配置项 | 示例 |
|--------|------|
| CD2 挂载路径 | `/115open` |
| 关联账号 | `我的115` |
| 显示标签 | `我的115` |

配置完成后，MediaTidy 中使用的路径会变成：

```text
/我的115/
```

例如：

```text
/我的115/downloads
/我的115/电影
/我的115/电视剧
```

验收标准：

- 文件管理器能看到 `/我的115/`。
- 点进去能看到网盘里的文件和目录。

## 第八步：配置 TMDB

自动整理需要 TMDB 来识别电影和电视剧。

1. 打开「系统设置」。
2. 找到「TMDB 配置」。
3. 一般情况下可以先使用内置默认 API Key。
4. 如果你有自己的 TMDB API Key，也可以填入自己的。
5. 如果 TMDB 访问不稳定，在「代理配置」里设置 HTTP 或 SOCKS5 代理。

验收标准：

- 系统设置保存成功。
- 解析测试或整理任务能搜索到正确的影视信息。

## 第九步：先做一次文件管理测试

在创建自动任务前，先确认 MediaTidy 能正常操作文件。

1. 打开「文件管理」。
2. 进入 `/我的115/`。
3. 找一个测试目录。
4. 尝试新建一个空文件夹。
5. 再删除这个空文件夹。

验收标准：

- 能正常进入目录。
- 能新建和删除测试文件夹。
- 如果这里失败，先不要创建整理方案，优先检查 CD2、115 Cookie 和路径映射。

## 第十步：创建第一个自动整理方案

适合想把下载目录自动整理成标准媒体库的人。

示例目录规划：

```text
源目录：/我的115/downloads
目标目录：/我的115/媒体库
未识别目录：/我的115/未识别
```

操作步骤：

1. 打开「自动整理」。
2. 点击「新建方案」。
3. 方案名称填写 `默认整理`。
4. 源目录填写 `/我的115/downloads`。
5. 目标目录填写 `/我的115/媒体库`。
6. 未识别目录填写 `/我的115/未识别`。
7. 传输方式新手建议先选 `move` 或 `copy`。
8. 数据源新手建议先选 `cd2_list`。
9. 先开启「试运行」。
10. 保存方案。
11. 手动执行一次。

::: tip 新手建议
第一次一定先用「试运行」。确认识别结果、目标路径和命名规则都没问题，再关闭试运行执行真实整理。
:::

验收标准：

- 任务能跑完。
- 日志里能看到识别结果。
- 试运行时目标路径符合预期。
- 关闭试运行后，文件能移动或复制到目标目录。

## 第十一步：创建第一个 STRM 方案

适合想让 Emby / Jellyfin 播放云盘视频、不想把视频下载到本地的人。

示例目录规划：

```text
源目录：/我的115/媒体库
输出目录：/app/strm
```

操作步骤：

1. 打开「STRM 生成」。
2. 点击「新建方案」。
3. 方案名称填写 `默认 STRM`。
4. 源目录填写 `/我的115/媒体库`。
5. 输出目录填写 `/app/strm`。
6. 如果源目录是 115 网盘，数据源建议选「115 高速穿透」；如果不是 115 资源，再选「CloudDrive2 列表」。
7. 并发数先保持默认。
8. 保存方案。
9. 手动执行一次。

::: tip 数据源怎么选
「115 高速穿透」适合 STRM 任务，直接走 115 原生接口生成播放地址，速度和播放链路更合适。  
「CloudDrive2 列表」更适合通用文件扫描，稳定但不一定是 115 STRM 的最佳选择。
:::

执行完成后，宿主机目录里应该能看到 `.strm` 文件：

```text
/opt/mediatidy/strm
```

验收标准：

- 任务能跑完。
- 输出目录生成了 `.strm` 文件。
- `.strm` 文件目录结构和源目录基本一致。

## 第十二步：接入 Emby

如果你使用 Emby，可以把 STRM 输出目录添加到 Emby 媒体库。

1. 登录 Emby 管理后台。
2. 创建或编辑媒体库。
3. 添加 STRM 输出目录。
4. 如果 Emby 和 MediaTidy 在不同容器里，确认 Emby 也能访问同一个 STRM 目录。
5. 扫描媒体库。

如果需要 MediaTidy 自动刷新 Emby：

1. 在 Emby 后台进入「设置 → 高级 → API 密钥」。
2. 新建一个 API Key。
3. 回到 MediaTidy 的「Emby 管理」。
4. 添加 Emby 服务器地址和 API Key。
5. 点击测试连接。
6. 在整理方案或 STRM 方案里开启 Emby 刷新联动。

验收标准：

- Emby 能扫描到 `.strm` 文件。
- 点击播放时能正常拉起播放。
- MediaTidy 中 Emby 连接测试通过。

## 第十三步：确认没问题后再开启自动化

手动跑通后，再考虑自动化。

自动整理可以开启：

| 功能 | 新手建议 |
|------|----------|
| 定时任务 | 推荐先每 6 小时或每天一次 |
| 实时模式 | 新手优先用 `polling` |
| 并发数 | 先保持 1 |
| ffprobe 探测 | 需要详细命名或洗版时再开 |
| 洗版升级 | 确认规则稳定后再开 |

STRM 生成可以开启：

| 功能 | 新手建议 |
|------|----------|
| 定时任务 | 推荐每天凌晨跑一次 |
| 实时模式 | 新手优先用 `polling` |
| 清理孤儿 STRM | 确认源目录稳定后再开 |
| 强制覆盖 | 平时关闭，需要重建时再开 |

## 常见问题

### 打不开管理面板

先检查容器是否启动：

```bash
docker ps
docker logs --tail 100 mediatidy
```

常见原因：

- 授权码没有填写或填写错误。
- 端口 `2019` 被其他服务占用。
- 防火墙没有放行 `2019`。
- 容器启动后马上退出。

### 文件管理里看不到网盘

优先检查：

- CD2 是否正常运行。
- CD2 地址是否填对。
- CD2 账号密码或 Token 是否正确。
- 115 Cookie 是否过期。
- 路径映射里的 CD2 挂载路径是否和 CD2 里一致。

### Docker 里看不到 FUSE 挂载目录

确认 compose 里使用了 `:rslave`：

```yaml
- /CloudNAS:/CloudNAS:rslave
```

还要确认宿主机上的 `/CloudNAS` 本身能看到文件。

### 自动整理识别不出来

常见原因：

- 文件名太乱，缺少标题或年份。
- TMDB 无法访问。
- 代理没有配置好。
- 电影和电视剧类型混在一起，命名不规范。

可以先用「工具箱 → 解析测试」查看系统如何理解文件名。

### STRM 生成了但 Emby 播放不了

优先检查：

- Emby 是否能访问 STRM 输出目录。
- `.strm` 文件内容是否是可访问 URL。
- 115 Cookie 是否有效。
- MediaTidy 服务 URL 是否配置正确。
- Emby 容器和 MediaTidy 容器的路径映射是否一致。

### 触发 115 风控

先降低请求频率：

- 自动整理并发数保持 1。
- STRM 并发数降低。
- 增大 115 API 调用延迟。
- 避免频繁全量扫描大目录。

## 新手推荐配置

第一周建议使用保守配置，稳定后再加速。

| 项目 | 推荐值 |
|------|--------|
| 自动整理数据源 | `cd2_list` |
| 自动整理并发 | `1` |
| 自动整理实时模式 | `polling` 或关闭 |
| STRM 数据源 | 115 资源选「115 高速穿透」，非 115 资源选「CloudDrive2 列表」 |
| STRM 并发 | 默认值 |
| 115 API 延迟 | 保持默认，风控后再增大 |
| ffprobe | 先关闭 |
| 洗版升级 | 先关闭 |
| 强制覆盖 STRM | 先关闭 |

## 最小成功清单

部署完成后，按这个清单确认：

- MediaTidy 管理面板能打开。
- 管理员账号能登录。
- CD2 连接测试通过。
- 115 账号状态正常。
- 文件管理器能看到 `/我的115/`。
- TMDB 能正常搜索。
- 自动整理试运行结果正确。
- 至少一个测试文件真实整理成功。
- STRM 方案能生成 `.strm` 文件。
- Emby 能扫描并播放测试媒体。

完成这些，再去配置更多规则、缓存、通知、反代和自动化任务，会顺很多。
