# STRM 生成

STRM 生成功能将云端视频文件转换为 `.strm` 文件，供 Emby / Jellyfin 等媒体服务器直接播放，无需下载完整视频。

## 工作原理

`.strm` 文件是一个纯文本文件，内容是视频的播放地址（URL）。媒体服务器读取 `.strm` 文件后，直接通过该地址串流播放。

```
源目录（云盘）                   输出目录（本地）
├── 电影/                        ├── 电影/
│   └── 阿凡达.mkv      →       │   └── 阿凡达.strm
└── 电视剧/                      └── 电视剧/
    └── S01E01.mkv       →           └── S01E01.strm
```

## 创建 STRM 方案

进入「STRM 生成」页面，点击「新建方案」。可以创建多个方案处理不同目录。

### 基本配置

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **方案名称** | `name` | 自定义名称，便于在列表中识别 |
| **源目录** | `sourceDir` | 云盘上的视频目录（虚拟路径格式，如 `/我的115/影视`） |
| **输出目录** | `outputDir` | STRM 文件的本地输出路径（如 `/app/strm`） |
| **所属账号** | `accountId` | 关联的 115 账号 ID |

### 数据源

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **数据源** | `dataSource` | 遍历文件的方式 |
| **遍历模式** | `mode` | 遍历模式 |

| 界面选项 | 内部值 | 说明 | 适用场景 |
|----------|--------|------|----------|
| **115 高速穿透** | `download_api` | 通过 115 原生 API 获取文件列表和播放地址 | 115 网盘 STRM 任务推荐 |
| **CloudDrive2 列表** | `cd2_list` | 通过 CD2 API 列出目录内容 | 非 115 存储或需要通用稳定扫描 |
| **115 目录树导出** | `tree_export` | 使用 115 目录树导出功能批量获取 | 超大目录首次生成或重建 |

::: tip 推荐
如果源目录是 115 账号路径，STRM 任务优先选「115 高速穿透」。它更适合生成可播放的直链地址。  
如果源目录不是 115，或者只是想用 CD2 做通用目录扫描，再选「CloudDrive2 列表」。
:::

### URL 生成

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **CD2 URL 后缀** | `cd2URLSuffix` | 追加到 CD2 生成的下载 URL 后面的参数（一般不需要设置） |
| **PC 参数 Key** | `pcParamKey` | 115 PC 端下载地址的参数 key（高级设置） |
| **Account 参数 Key** | `accountParamKey` | 115 账号标识的参数 key（高级设置） |

### 文件过滤

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **排除关键词** | `excludeWords` | 文件路径包含这些关键词的文件跳过（如 `["sample", "extras"]`） |
| **视频扩展名** | `videoExts` | 自定义视频文件扩展名（留空则使用系统全局设置） |
| **元数据扩展名** | `metadataExts` | 自定义元数据文件扩展名（留空则使用系统全局设置） |

### 并发与性能

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **STRM 并发数** | `numWorkers` | 4 | 同时生成 STRM 文件的线程数 |
| **元数据并发数** | `metadataWorkers` | 2 | 同时同步元数据文件的线程数 |

### 覆盖与清理

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **强制覆盖** | `forceOverwrite` | 开启后重新生成所有 STRM 文件（关闭则增量生成，跳过已存在且内容相同的文件） |
| **清理孤儿 STRM** | `cleanupOrphans` | 源目录中已删除的视频，自动删除对应的 `.strm` 文件 |
| **清理孤儿元数据** | `cleanOrphanMetadata` | 源目录中已删除的视频，自动删除对应的元数据文件（字幕、NFO 等） |
| **清理源空目录** | `cleanSourceEmptyDirs` | 生成完成后清理源目录中的空文件夹 |

### 元数据同步

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **元数据模式** | `metadataMode` | 控制元数据文件的同步方式 |

STRM 生成过程中会同步处理与视频同名的元数据文件：

- **字幕文件**：.srt, .ass, .ssa, .sub 等
- **NFO 元数据**：.nfo
- **海报图片**：.jpg, .png, .tbn 等

## 定时任务

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **Cron 表达式** | `cronExpr` | 标准 5 位 Cron 表达式 |
| **启用定时** | `cronEnabled` | 开关定时任务 |

```
0 3 * * *      # 每天凌晨 3 点全量生成
0 */4 * * *    # 每 4 小时执行一次
```

## 实时模式

当源目录有新文件时自动生成对应的 `.strm` 文件。

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **实时模式** | `realtimeMode` | `off` / `webhook` / `listener` / `polling` |
| **延迟时间** | `realtimeDelay` | 单位秒，检测到新文件后等待多久再处理 |

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `off` | 关闭实时生成 | 仅使用定时任务或手动触发 |
| `webhook` | 配合 CD2 的 Webhook 功能，文件变化时实时通知 | CD2 已配置 Webhook |
| `listener` | 持续监听源目录的文件系统变化事件 | 需要 FUSE 挂载 |
| `polling` | 定期扫描源目录检测新增文件 | 通用方案，兼容性最好 |

---

## 增量与覆盖

- **增量生成**（默认）：跳过已存在的 `.strm` 文件
- **内容比对**：即使文件存在，也会比对 STRM 文件内容，仅在播放地址变化时更新
- **强制覆盖**：开启 `forceOverwrite` 后，重新生成所有 STRM 文件

## Emby 刷新联动

STRM 生成完成后可自动触发 Emby 媒体库刷新：

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **启用 Emby 刷新** | `embyRefreshEnabled` | 生成完成后自动刷新 Emby 媒体库 |
| **刷新服务器** | `embyRefreshServers` | 指定刷新哪些 Emby 服务器（留空 = 全部） |
| **刷新延迟** | `embyRefreshDelay` | 生成完成后等待多少秒再刷新（`0` = 立即） |
| **路径映射** | `embyPathMappings` | 本地路径 → Emby 路径的映射 |

路径映射配置同[自动整理 - Emby 刷新联动](/features/organize#emby-刷新联动)。

## STRM 神医 FF 缓存

MediaTidy 的「STRM 神医」会读取 Emby 媒体源信息，用于恢复或预热 STRM 播放所需的媒体流数据。新版远程 FF 缓存服务为神医单独提供 `strm_assistant_ff` 缓存域：

- 先查本地 sidecar JSON 和本地缓存
- 未命中时按文件 SHA1 查询远程神医缓存
- 远程命中后直接恢复 Emby 媒体源信息
- 未命中时再从 Emby 提取，并可批量上传到远程服务

这能显著减少大规模 STRM 库迁移、重建、批量预热时的重复 Emby 查询，让库内媒体信息恢复得更快。工作原理见 [FF 缓存](/features/ff-cache-server)。

## STRM 神医整理池定位

开启自动整理池后，整理完成的真实媒体路径会按当前 STRM 方案和 Emby 路径映射转换为对应的 `.strm` 路径，再优先定位 Emby 中的 STRM 媒体项。例如映射后的 `/media/电影/Movie.mkv` 会先查询 `/media/电影/Movie.strm`。

- 如果媒体库直接暴露真实视频文件，系统会继续使用映射后的原媒体路径作为兼容回退
- 整理池任务真正执行时会重新读取当前 STRM 方案和 Emby 路径映射，并把重算结果写回任务记录；任务排队后修改配置不会继续使用旧路径
- 在 Linux 等区分大小写的输出文件系统上，仅大小写不同且能够实际并存的目标允许生成；在 Windows 等大小写不敏感环境，或文件名清理后确实落到同一目标时，仍会判定冲突并停止覆盖
